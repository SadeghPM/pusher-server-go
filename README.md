# Pusher Clone (Go)

A lightweight, in-memory Pusher alternative written in Go, specifically designed to be compatible with Laravel's broadcasting system using the standard Pusher Protocol v7.

## Features

- Fully compatible with Pusher Protocol v7
- Supports public and private channels
- Event broadcasting from Laravel
- In-memory state management (suitable for single-server setups)
- Implements Pusher signature validation for API requests and private channel subscriptions

## Setup and Installation

1. Make sure you have Go installed.
2. Clone this repository.
3. Install dependencies:
   ```bash
   go mod tidy
   ```
4. Copy the `.env.example` file to `.env`:
   ```bash
   cp .env.example .env
   ```
5. Update the `.env` file with your credentials (or keep the defaults for local testing).

## Running the Server

```bash
go run main.go
```

The server will start on the port specified in your `.env` file (default: 8080).

## Laravel Configuration

To use this clone in your Laravel application, update your `config/broadcasting.php` file:

```php
'pusher' => [
    'driver' => 'pusher',
    'key' => env('PUSHER_APP_KEY'),
    'secret' => env('PUSHER_APP_SECRET'),
    'app_id' => env('PUSHER_APP_ID'),
    'options' => [
        'host' => '127.0.0.1', // Your Go server's IP
        'port' => 8080,        // Your Go server's Port
        'scheme' => 'http',
        'encrypted' => false,
        'useTLS' => false,
    ],
],
```

Ensure the credentials in your Laravel `.env` match the ones in your Go server's `.env`.

```env
BROADCAST_DRIVER=pusher
PUSHER_APP_ID=my-app-id
PUSHER_APP_KEY=my-app-key
PUSHER_APP_SECRET=my-app-secret
```

## Supported Channels
- **Public channels:** Works out of the box.
- **Private channels:** Supported. The clone will verify the HMAC signature generated by your Laravel application during the `pusher:subscribe` event.
